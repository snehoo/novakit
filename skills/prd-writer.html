Problem Statement
Outcome-first framing. Problem only — no solution leak.
Current state: Admins manually export CSVs from three separate dashboards, merge them in Sheets, and email the result to stakeholders — typically taking 45–90 minutes each Monday.

Pain point: The process breaks when any one export format changes. 3 of the last 8 Mondays resulted in an error that delayed the report by half a day.

Why now: Q3 headcount growth means the manual process will hit its limit within 60 days.
Under 100 words. Scannable in 10 seconds. No solutions in this section.
User Story + GWT Criteria
Behaviour-first. QA-testable without asking the PM.
US-02: Scheduled report delivery
Priority: Must Have

As an admin, I want weekly reports delivered automatically to stakeholder inboxes, so that I don't have to manually compile and send them.

Scenario 1 — Successful delivery
Given a report schedule is configured and active
When Monday 08:00 UTC arrives
Then stakeholders receive the report within 5 minutes

Scenario 2 — Delivery failure
Given an email address in the list is invalid
When the report is dispatched
Then the admin receives a failure notification within 10 minutes
Every Must Have story has a failure scenario. Acceptance criteria need no PM clarification.
Edge Cases
Five categories covered. Assumptions flagged explicitly.
Empty state: No data in selected date range → show "No data available for this period" with a date-reset CTA
Auth failure: User session expires mid-export → redirect to login, restore export context on return
Network error: Export job times out after 30s → surface retry button, do not silently fail
Concurrent action: Two admins trigger export simultaneously → queue second request, do not duplicate output
Data limit: Export exceeds 50,000 rows → prompt user to apply date filters, block oversized export [ASSUMED — please confirm limit]
Minimum 5 edge cases per PRD. Assumptions flagged as [ASSUMED] for PM sign-off.
Non-Functional Requirements
Quantified. Testable. No "fast" or "secure" allowed.
Performance: Export job completes in <8s for datasets up to 10,000 rows at p95
Security: Exported files encrypted in transit (TLS 1.3); download links expire after 24 hours
Reliability: Scheduled reports must achieve 99.5% on-time delivery; failures trigger admin alert within 10 min
Accessibility: Export controls meet WCAG 2.1 AA
Compliance: GDPR — exported data must not include fields flagged as PII unless admin has data-export role [NEEDS TECH REVIEW]
Every NFR is measurable. Items needing engineering input flagged as [NEEDS TECH REVIEW].
