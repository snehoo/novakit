## Planning Rubric

1. **What is the product?** NovaKit — a marketplace of Claude Skills (LinkedIn posts, cold email, pitch decks, SEO, etc.), $5-$19 each, bundles from $15. Tagline on site: "Claude Skills that actually work." Core mechanic per `package.json` description: "Living skills that do live trend research before every output."
2. **Who is this video for?** A power Claude user / creator who already has skills installed and uses them constantly — not someone discovering Claude Skills for the first time. The pain is specific to heavy users: after enough runs, every skill feels like it's reusing the same scaffolding — same hook shape, same structure, same vibe — because a static prompt has no new information run to run.
3. **What's the single claim worth a video?** NovaKit skills check live trends before every single run, ask 3 narrowing questions, and run a quality gate that rewrites weak drafts — so the same skill run today vs. in six months gives a different (better-fitted) result, because the underlying research changed and the skill knew.
4. **What's the actual proof, not a vibe?** Site UI: live "Trend research: live" badge, the 3-question narrowing flow, the timeline showing "Live trend research → Delivering output, trend-stamped," and FAQ copy: *"Run the same skill today and in six months — you'll get different results because what works will have changed, and the skill will know."*
5. **What's the contrast?** Stale, identical output (typing the same prompt into Claude directly, getting the same shape every time) vs. NovaKit's per-run live research loop.
6. **Tone?** Closest preset: `default` (playful, clean, postable) — but dialed for speed: fast-cut, poppy, high-energy editing. Per the user's explicit instruction, the *content* stays grounded and specific — no trend-bait gimmicks, no "see what's viral now" theatrics. The speed is in the cutting, not in the message chasing relevance.
7. **What's the hook?** Open on the exact pain a power user recognizes instantly: a feed of outputs that all look the same. Cold open, no warm-up.
8. **What's the visual centerpiece?** The actual NovaKit timeline UI — "Live trend research" step ticking, the narrowing-question selector, the quality-gate rewrite moment.
9. **What's the CTA?** novakit.tech, skills from $5.

## Creative Angle

**"Same prompt. Same skill. Same... everything. Until it isn't."**

Open on the exact fatigue a heavy Claude-skill user feels: outputs that have started to blur together. Cut hard into the fix — NovaKit skills don't run on a static prompt, they research live before every single output. Show the receipts (narrowing questions, quality gate, trend-stamp), then land the punchline: run the same skill in six months, get a different, better-fitted result — because the trend changed and the skill knew. No "trending now" theatrics, no meme format — just fast, confident, specific cuts.
